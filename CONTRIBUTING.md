# Contributing Guide

このリポジトリへの貢献ありがとうございます。  

## 注意点

作者は卒業済みのため、作者本人による動作確認はできません。
PR には動作確認の動画やスクリーンショットを添付していただけると助かります。

## できること

だいたいの変更は歓迎します。例えば：

- 不具合修正
- 学務情報システムの仕様変更への追従
- 使い勝手の改善
- メンテナンス（依存更新、軽微な整理）

## 開発の流れ

1. このリポジトリを Fork します
2. 作業用ブランチを作成します
3. 変更を小さく保ちつつ開発します（1 PR = 1 目的）
4. 動作確認・チェックを行い、PR を作成します
5. Approve されたら PR Author がマージします

## 開発

- `bun run dev`  
  ローカル開発サーバを起動（アクセス時にビルド）

- `bun run build`  
  配布用ファイルを `dist/` に生成

- `bun run check`  
  Biome による lint / format の確認

- `bun run check-write`  
  lint / format の自動修正
